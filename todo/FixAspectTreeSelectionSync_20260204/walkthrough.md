# 演练 - 修复工艺功能树选择联动问题

## 任务概览
用户发现在“工艺功能树”中，选中一个系统的节点（如“弱电消防”）时，另一个系统（如“照明”）中相同的设备会被自动选中。这是因为两个树节点在后端使用了相同的 ID（指向同一个设备），当前端树组件使用该 ID 作为唯一 Key 时，会导致状态同步。

修复方案是为前端树的每个节点生成一个唯一的 `uitreeId`，从而解耦它们的选中状态。

## 变更内容

### 1. `AspectTreePanel.vue`
- **生成唯一 UI ID**：在接收到后端数据并处理节点树（`processNodes`）时，为每个节点新增 `uitreeId` 字段（格式：`originalId_randomString`）。
- **更新树配置**：将 `el-tree-v2` 的 `node-key` 属性从 `id` 改为 `uitreeId`。
- **更新点击逻辑**：`handleNodeClick` 现在使用 `uitreeId` 来反转节点的勾选状态。

## 验证结果

### 预期行为
1.  **独立选择**：现在选中“弱电消防系统”下的某个探测器，不会再导致“照明系统”下的同名探测器被选中。
2.  **功能完整**：尽管 UI 上是分开的，但选中的节点仍然包含正确的原始 ID，因此“模型高亮”和“追溯”功能依然能正确找到对应的物理构件。
