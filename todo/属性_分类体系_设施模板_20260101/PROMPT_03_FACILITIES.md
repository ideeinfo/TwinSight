# Prompt 03: Facilities

```text
Page to generate: Facilities

Use the existing TwinSight UI design language.

Global application shell requirements:
- Add a top navigation bar with Home, Facilities, and Manage
- Facilities is active on this page

Page purpose:
- This page is used to create, list, edit, and manage facilities
- It is the central facility management page
- It is not the metadata management page

Main layout:
- Top navigation
- Page header with title "Facilities"
- Primary action button for creating a facility
- Search and filter controls
- Facility list area

Facility list requirements:
- Support a card grid or dense table/list hybrid
- Each facility should show:
  - name
  - project name
  - owner
  - address
  - thumbnail in a 16:9 visual area
- Provide actions:
  - open facility
  - edit
  - delete or more menu

Design direction:
- Practical operations screen
- Clean dark enterprise layout
- Strong scanability
- Minimal decorative elements

Important distinction:
- This page manages facilities as objects
- It does not directly manage properties, classifications, or templates

Deliverable:
- Desktop-first facility management page
- Include realistic empty state and populated state behavior in the composition
```
