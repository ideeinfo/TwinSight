## 第二部分：AI数据层 - 数据的智能化全生命周期 (4-5页)

### 第4页：AI驱动的数据提取

**场景：从PDF文档到结构化数据**

**传统方式**：
- 手动录入 → 耗时、易错、不可扩展

**AI原生方式**：
```
PDF文档 (设备说明书/规格书)
    ↓
AI文档解析 (OCR + NLP)
    ↓
结构化数据提取
    ↓
自动入库 (资产规格库)
```

**技术亮点**：
- 支持PDF、Word、Excel、图片等多种格式
- 智能识别表格、参数、型号
- 自动关联到对应资产/空间

**案例**：
> 上传一份空调设备说明书PDF → AI自动提取300+参数 → 填充到资产规格表

### 第6页：RDS数据智能生成

**什么是RDS？**
- **R**elational **D**ata **S**ync - 关系数据同步
- 将非结构化数据转化为结构化关系数据

**AI生成流程**：
```
原始文档 (PDF/图片/扫描件)
    ↓
AI文档理解 (多模态大模型)
    ↓
实体识别 (资产/空间/关系)
    ↓
关系抽取 (连接关系)
    ↓
RDS数据生成 (可直接导入数据库)
```

**应用场景**：
- 建筑竣工图纸 → 自动生成空间关系
- 设备清单表格 → 自动生成资产树
- 维保记录 → 自动生成资产生命周期数据

---

### 第7页：IoT数据的AI预处理

**挑战**：IoT数据量大、噪声多、缺失值

**AI原生处理**：
```
原始IoT时序数据
    ↓
AI异常检测 (识别传感器故障)
    ↓
AI数据补全 (基于时序预测模型)
    ↓
AI特征提取 (降维、模式识别)
    ↓
高质量时序数据库
```

**智能标注**：
- AI自动识别数据模式（如：周期性、趋势）
- 自动生成数据质量报告
- 智能推荐采样频率

## 第四部分：AI决策层 - MCP协议与系统协作 (3-4页)

### 第12页：MCP协议 - 让AI系统对话

**什么是MCP？**
- **M**odel **C**ontext **P**rotocol - 模型上下文协议
- 让不同AI系统之间可以交换信息和协作

**TwinSight作为MCP Server**：
```
┌─────────────────────────────────────┐
│         其他AI系统                  │
│  - Claude (Anthropic)               │
│  - ChatGPT (OpenAI)                 │
│  - 自定义AI Agent                   │
└──────────────┬──────────────────────┘
               │ MCP协议
               ↓
┌─────────────────────────────────────┐
│       TwinSight MCP Server          │
├─────────────────────────────────────┤
│  提供的能力 (Tools):                │
│  - query_asset: 查询资产信息        │
│  - query_space: 查询空间信息        │
│  - query_timeseries: 查询时序数据  │
│  - query_document: 检索文档         │
│  - ai_analysis: 触发AI分析          │
│  - export_data: 导出数据            │
└─────────────────────────────────────┘
```

### 第13页：MCP应用场景

